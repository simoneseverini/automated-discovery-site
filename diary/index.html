Rigor and Generation

This entry is based on the survey arXiv:2508.20825.

Computer-assisted graph theory has evolved from the controversial "brute force" of the 1976 Four Color Theorem to a rigorous discipline. Modern approaches prioritize correctness through formal verification (e.g., using the Rocq proof assistant) and independent algorithm reimplementation.

The fundamental engine of the field is Exhaustive Generation—producing every non-isomorphic graph in a specific class (e.g., cubic, planar) up to a certain order. The central challenge is Isomorphism Rejection: preventing the generation of structurally identical copies without the memory cost of storing every graph found. The survey highlights sophisticated solutions to this:

  • Orderly Generation: Introduced by Faradžev and Reed, this method generates graphs such that only those with a specific "canonical" labeling are accepted. Because validity is determined by the current graph's structure alone, it requires no historical memory.
  • Canonical Construction Path: Used by the standard tool nauty, this method assumes graphs are built recursively (e.g., adding edges). To avoid duplicating work via different build orders (a problem known as pseudosimilarity), the algorithm defines a "canonical reduction". A graph is accepted only if it was generated via the exact reverse of its canonical reduction.

Searchable Knowledge

The field has moved beyond static lists of graphs to dynamic databases like the House of Graphs. Unlike standard repositories, these allow researchers to search using inequalities of invariants (e.g., searching for graphs where chromatic number > clique number). This facilitates the discovery of connections between unrelated problems by identifying common counterexamples.

Algorithmic Paradigms

Exact & Algebraic Methods
  • MILP: Used for exact structural problems (e.g., Turán numbers). Duality provides mathematical certificates of optimality.
  • SDP & Flag Algebras: Transform combinatorial subgraph density problems into algebraic inequalities to provide rigorous asymptotic bounds (e.g. Ramsey numbers).
Logic & Machine Learning
  • SAT Solvers: Used for boolean constraints. Modern solvers produce "proof logs" that formally verify non-existence of counterexamples.
  • Machine Learning: Approaches like FunSearch utilize LLMs to generate code that searches for "witnesses," evolving algorithms rather than just data.

Resource-Constrained Graph Construction

Fix integers $k,m\ge 0$. We consider a construction process occurring over time steps $t=1,2,\dots,n$. At each time $t$, a new vertex $v_t$ is created. Alice sends an instruction $x_t\in\{0,1\}^k$. Bob maps this instruction to a stored state \[ \ell(t) \;=\; \alpha(x_t) \in \Sigma_m, \qquad \text{where } \Sigma_m := \{0,1,\dots,2^m-1\}. \] The state $\ell(t)$ is permanently attached to $v_t$.

The crucial constraint is that the adjacency of $v_t$ to all earlier vertices must be determined solely from the stored states. Concretely, Bob fixes a decoder function once and for all: \[ \delta: \Sigma_m \times \Sigma_m \to \{0,1\}. \] Upon creating $v_t$, edges are added according to the rule: \[ \{v_i, v_t\} \in E \iff \delta(\ell(i), \ell(t)) = 1 \qquad (\text{for } i < t). \]

Definition (Resource model $\mathcal{R}(k,m)$). A graph $G$ is constructible in $\mathcal{R}(k,m)$ if there exist functions $\alpha, \delta$ and instructions $(x_t)$ producing $G$ via this protocol.

To characterize these graphs, we recall lettericity. An ordered graph is a $(\Sigma,D)$-letter graph if edges correspond to pairs of letters in $D$. The lettericity is the minimum alphabet size required.

Theorem. A graph $G$ is constructible in $\mathcal{R}(k,m)$ if and only if its lettericity satisfies: \[ \text{lett}(G) \le 2^{\min\{k,m\}}. \]

Proof Sketch.

The effective alphabet available to the system is bounded by Alice's instruction bandwidth ($2^k$) and Bob's memory ($2^m$). Thus, $K = \min\{2^k, 2^m\}$.

(Forward): If $G$ is generated, the set of states actually used acts as the alphabet $\Sigma$, and the decoder acts as the relation $D$. Thus $\text{lett}(G) \le K$.
(Reverse): If $\text{lett}(G) \le K$, we can encode the letters of the graph into the available binary instructions and memory states, constructing the graph exactly. $\square$

  • M. Petkovšek, "Letter graphs and well-quasi-order," Discrete Math., 2002.

The Missing Moore Graph

A Moore graph of diameter $2$ and degree $k$ is a regular graph with the theoretical maximum number of vertices allowed by the degree-diameter bound. The total number of vertices is $n = k^2 + 1$. Hoffman and Singleton (1960) proved that for such a graph to exist, $k$ must belong to $\{2, 3, 7, 57\}$.

  • $k=2$: The pentagon ($5$ vertices).
  • $k=3$: The Petersen graph ($10$ vertices).
  • $k=7$: The Hoffman-Singleton graph ($50$ vertices).

The case $k=57$ ($3250$ vertices) has been open for 60 years. In The missing Moore graph as an optimization problem (2023), Smith and Montemanni attack this using massive parallel hill-climbing. They converged to a solution with a deficit of $41,482$ edges, suggesting non-existence, but the rugged landscape makes it a fascinating testbed for evolutionary algorithms.

Markov Triples & The Fibonacci Spine

A Markov triple is a triple of positive integers $(x,y,z)$ satisfying $x^2+y^2+z^2 = 3xyz.$ Navigating the solutions via Vieta jumping generates the Markov tree. The question is which Markov triples have a Fibonacci number as their largest entry?

If we fix $x=1$, the equation becomes $y^2+z^2+1 = 3yz$. The positive solutions are exactly $(1, F_{2n-1}, F_{2n+1})$, forming a "spine" in the tree:

  • $(1,F_1,F_3)=(1,1,2)$
  • $(1,F_3,F_5)=(1,2,5)$
  • $(1,F_5,F_7)=(1,5,13)$
Figure: The Fibonacci spine (red) and the side branches (blue). Once you leave the spine, the magnitudes accelerate so fast that hitting a Fibonacci value is highly unlikely.

Shannon Capacity of $C_7$

The Shannon capacity of a graph $G$ is defined as $\Theta(G) = \lim_{d \to \infty} (\alpha(G^d))^{1/d}.$ The best lower bound for $C_7$ is $\Theta(C_7) \geq \sqrt[5]{367} \approx 3.2578$. (arXiv:1808.07438).

Algorithm Description

The algorithm searches for the Maximum Independent Set (MIS) in the graph $G = C_7^n$ using a Randomized Iterated Local Search strategy. It combines greedy construction (filling the set with valid vertices) with a "Ruin and Recreate" perturbation step to escape local optima.
